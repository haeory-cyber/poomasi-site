      AI 운영 규칙

      작업 전
      • git pull 먼저 -- 항상 최신 파일 기준
      • 수정 전 백업 필수 -- 파일명.bak_YYYYMMDD
      • 배포는 poomasi-site-git에서만
      작업 중
      • 읽기/조사/테스트 → AI가 직접 (후니님한테 안 떠넘김)
      • 파일 수정/배포 → 확인받고 실행
      • 한 건씩 수정 → 확인 → 다음 건
      절대 금지
      • scheduler.py, netlify_deploy.py, netforce.py 실행 금지
      • git add -A 금지 -- 파일 개별 지정만
      • 승인 없이 push 금지
      • CLAUDE.md, mcp_server.py 무단 수정 금지
      배포 흐름
      • ① 레포 검증 + git pull
      • ② 자동 백업 (.bak)
      • ③ 수정
      • ④ 민감파일 스캔
      • ⑤ diff 출력 → 후니님 승인 대기
      • ⑥ git add (개별) + commit
      • ⑦ push → ⑧ 헬스체크 → ⑨ 실패 시 롤백

      단축 명령어

      이렇게 말하면 실행되는 것
      "총괄 읽어" SHARED_MEMORY.md (AI간 공유 메모리) 조회
      "인수인계 읽어" HANDOVER_MIR.md (최근 작업 인수인계) 조회
      "체크리스트 봐" 지원사업 체크리스트 조회
      "레슨 읽어" LESSONS_INDEX.md (학습 기록) 조회
      "배포해" /deploy 스킬 9단계 실행
      "마무리해" /session-handover 인수인계 저장
      "스킬업" 대화 맥락 분석 → 레슨/메모리/체크리스트/스킬 등 적절한 곳에 자동 기록
